# OpenMontage：從 Git Clone 到成功渲染影片

本文件整理自官方 `README.md`、`Makefile`、`.env.example` 與專案慣例，目標是讓你在全新環境中，從 `git clone` 一路做到**可驗證的影片輸出**（零 API 金鑰亦可）。

---

## 1. 建議環境與心態

| 項目 | 說明 |
|------|------|
| **作業系統** | **Linux** 或 **macOS** 為首選。若使用 **Windows**，強烈建議在 **WSL2（Ubuntu）** 內操作本專案，以避免 Node／路徑／編碼與 Remotion 相關問題。 |
| **網路** | 首次安裝需下載 Python 套件、npm 套件，以及 Remotion 可能自動下載的瀏覽器二進位檔。 |
| **磁碟** | 預留足夠空間給 `.venv`、`node_modules`、npx 快取與輸出影片。`make setup` 會嘗試預熱 HyperFrames（約數十 MB 等級）。 |

OpenMontage 是 **「由 AI 編碼助理讀檔並執行工具」** 驅動的製片系統；下列安裝完成後，你仍需在 Cursor／Claude Code 等環境中請助理依 pipeline 執行。本指南的「渲染成功」以專案內建的 **零金鑰 Remotion 示範** 為可重現的驗收標準。

---

## 2. 系統必備（請先裝在 OS 層）

以下為 **渲染與工具鏈** 通常必須具備的程式，請依你的發行版或平台安裝。

### 2.1 Git

用於 clone 儲存庫。

### 2.2 Python **3.10 或以上**

- 官方建議來源：[python.org](https://www.python.org/downloads/)
- Ubuntu 等可使用 `python3`；專案透過 `make setup` 會在專案根目錄建立 **`.venv`**，避免系統 PEP 668「外部管理環境」限制。

### 2.3 Node.js **18 或以上**（含 **npm**、**npx**）

- 官方建議來源：[nodejs.org](https://nodejs.org/)
- Remotion CLI 透過 `npx remotion` 呼叫，故 **npx 必須在 PATH 上**。

### 2.4 FFmpeg

後製、轉檔、燒字幕、音訊分析等多數工具會依賴 FFmpeg。

- **macOS（Homebrew）**：`brew install ffmpeg`
- **Debian／Ubuntu**：`sudo apt install ffmpeg`
- 其他平台請參考 [ffmpeg.org/download.html](https://ffmpeg.org/download.html)

### 2.5 GNU Make（選用但強烈建議）

專案提供 `Makefile` 一鍵安裝。若沒有 `make`：

- **Debian／Ubuntu**：`sudo apt install make`
- 或改用手動步驟（見第 4 節「無 make 時」）。

### 2.6 Remotion 渲染用的 Chromium（多數情況自動處理）

執行 `npx remotion render` 時，Remotion 通常會自動取得可用的 Chrome／Chromium。若在無頭伺服器或極簡環境失敗，請參考 [Remotion 官方文件](https://www.remotion.dev/docs/) 中關於 **Linux 依賴** 與 **`CHROME_EXECUTABLE_PATH`** 的說明，改指向本機已安裝的 Chrome／Chromium。

## 4. 一鍵安裝：`make setup`

在專案根目錄執行：

```bash
make setup
```

`make setup` 會依序完成（節錄自 `Makefile`）：

1. 建立 **`.venv`** 並升級 `pip`
2. **`pip install -r requirements.txt`**（核心 Python 依賴）
3. **`cd remotion-composer && npm install`**（Remotion 作曲器）
4. 嘗試 **`pip install piper-tts`**（失敗時略過，僅影響離線 TTS，雲端 TTS 仍可用）
5. 嘗試 **預熱 HyperFrames**（`npx hyperframes`；失敗時第一次渲染可能較慢）
6. 若根目錄尚無 **`.env`**，則 **`cp .env.example .env`**

完成後你應具備：Python 工具環境、Remotion 專案依賴、可編輯的 `.env`。

### 4.1 無 `make` 時的手動對等指令

在專案根目錄：

```bash
python3 -m venv .venv
source .venv/bin/activate   # Windows cmd: .venv\Scripts\activate.bat ／ PowerShell: .venv\Scripts\Activate.ps1
pip install --upgrade pip
pip install -r requirements.txt
cd remotion-composer && npm install && cd ..
pip install piper-tts       # 可選；失敗可略過
test -f .env || cp .env.example .env
```

**Windows 注意**：若 `npm install` 出現 `ERR_INVALID_ARG_TYPE`，README 建議改用：

```bash
npx --yes npm install
```

（在 `remotion-composer` 目錄內執行。）

## 5. 環境變數設定（`.env`）

安裝完成後，編輯專案根目錄的 **`.env`**（由 `.env.example` 複製而來）。

- **僅要驗證「能渲染」**：可維持空白或只放註解，**不必填任何 API 金鑰** 即可跑第 6 節的示範渲染。
- **要解鎖雲端圖像／影片／語音等**：依 `.env.example` 內註解填入對應變數，例如：`FAL_KEY`、`GOOGLE_API_KEY`、`ELEVENLABS_API_KEY`、`OPENAI_API_KEY`、`PEXELS_API_KEY` 等。

各變數用途以 `.env.example` 內說明為準；完整供應商對照可另見 `docs/PROVIDERS.md`（若儲存庫中有提供）。

## 6. 驗收：零 API 金鑰渲染成功（Remotion Demo）

專案內建 **`render_demo.py`**，會使用 `remotion-composer` 內的示範 props 呼叫：

`npx remotion render ... --codec h264`

### 6.1 啟用虛擬環境後執行

```bash
cd /你的/OpenMontage/路徑
source .venv/bin/activate
python render_demo.py --list    # 列出可用 demo 名稱
make demo                        # 或：python render_demo.py（渲染全部 demo）
```

### 6.2 輸出位置

預設輸出目錄：

```text
projects/demos/renders/
```

若該目錄出現對應的 **`.mp4`**，即表示 **Clone → 依賴安裝 → Remotion 渲染** 鏈路成功。

### 6.3 單獨手動渲染（對照用）

在已 `npm install` 的 `remotion-composer` 下亦可：

```bash
cd remotion-composer
npx remotion studio              # 預覽／除錯用
npx remotion render src/index.tsx Explainer out/video.mp4   # 需自行準備 props 等參數時參考 package.json scripts
```

## 7. HyperFrames 執行環境（HTML／GSAP 渲染）

若你會使用 HyperFrames 相關流程，建議在安裝後執行：

```bash
make hyperframes-doctor
```

此目標會透過 Python 工具探測 **Node、ffmpeg、npx** 與 HyperFrames CLI 是否就緒。亦可手動更新 npx 快取：

```bash
make hyperframes-warm
```

## 8. 選用：本機 GPU 影片生成

若你有 **NVIDIA GPU** 且要啟用文件所述之本機擴散模型路徑：

```bash
make install-gpu
```

並於 `.env` 設定（範例，實際以官方 README／`.env.example` 為準）：

```env
VIDEO_GEN_LOCAL_ENABLED=true
VIDEO_GEN_LOCAL_MODEL=wan2.1-1.3b
```

`make install-gpu` 會安裝 `requirements-gpu.txt` 並額外安裝 `diffusers`、`transformers`、`accelerate` 等；**體積與編譯依賴較大**，僅在需要本機生成時再裝。

## 9. 開發者額外套件（非一般渲染必須）

若你要跑測試或完整開發環境：

```bash
make install-dev
```

（使用 `requirements-dev.txt`。一般「製片並渲染」不必須。）

## 10. 常見問題速查

| 現象 | 建議處理 |
|------|----------|
| `make: command not found` | 安裝 `make`，或改用手動指令（第 4.1 節）。 |
| `python3: command not found` | 安裝 Python 3.10+，或確認 PATH。 |
| `node`／`npm`／`npx` 找不到 | 安裝 Node 18+ 並重新開啟終端機。 |
| `ffmpeg` 找不到 | 安裝 FFmpeg 並確認 `ffmpeg -version`。 |
| `npm install` 在 Windows 失敗 | 依 README 使用 `npx --yes npm install`；或改用 **WSL2**。 |
| `piper-tts` 安裝失敗 | 可略過；不影響 Remotion demo；僅離線 Piper TTS 不可用。 |
| Remotion 渲染報 Chrome／Chromium 錯誤 | 安裝系統 Chromium 或設定 `CHROME_EXECUTABLE_PATH`；查 Remotion 官方 Linux 依賴文件。 |
